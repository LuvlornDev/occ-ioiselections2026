🔁 Interactive Problems
Problems where your program talks back and forth with the judge

What is an Interactive Problem?

Unlike standard problems, interactives have a hidden state (a number, a sequence, a graph). Your program must query the judge and use the answers to figure out the answer. The judge responds in real time.

Your program Judge (hidden: N=42) ────────────────────────────────────────── ? 50 → "lower" ? 25 → "higher" ? 37 → "higher" ? 44 → "lower" ! 42 → "correct ✓"

⚠️ The Most Critical Rule: Flush!

After every output, you MUST flush stdout. Without it, your output sits in a buffer and the judge never sees it — you get TLE or WA.

C++
// Method 1: endl (flushes automatically)
cout << answer << endl;

// Method 2: explicit flush (faster)
cout << answer << '\n' << flush;

// Method 3: at program start (safest)
cout.sync_with_stdio(false);
cin.tie(&cout); // ties cin to cout, auto-flushes before read

Template: Guess the Number (Binary Search style)

C++
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(&cout); // auto-flush before every cin

    int lo = 1, hi = 1000;

    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        cout << "? " << mid << endl; // endl flushes!

        string resp;
        cin >> resp;

        if (resp == "lower")       hi = mid - 1;
        else if (resp == "higher")  lo = mid + 1;
        else break; // "correct"
    }
    cout << "! " << lo << endl;
    return 0;
}

Common Patterns in Interactives

Binary Search queries — find a value in [L, R] in O(log N) queries.

Comparison queries — ask "is A[i] > A[j]?" to sort or find max.

Bitwise queries — ask about individual bits to reconstruct a number.

Graph queries — ask "is there an edge between u and v?" to discover hidden graph structure.

Tip: Always check the query limit! If you have N=1000 and 10 queries, you need binary search. If you have N² queries, brute force may work.
🧩 Dynamic Programming
Break hard problems into overlapping subproblems with optimal substructure

The DP Mindset

DP = memoized recursion or bottom-up table filling. Ask yourself: "What is the state? What is the transition?"

  1. Define state: what variables fully describe a subproblem? (e.g., dp[i] = answer for first i elements)
  2. Define base case: what's trivially known? (e.g., dp[0] = 0)
  3. Write the transition: how does dp[i] depend on smaller states?
  4. Determine order: fill from smaller → larger states.

Example 1: Fibonacci (simplest DP)

C++
// dp[i] = i-th Fibonacci number
// Transition: dp[i] = dp[i-1] + dp[i-2]
vector<long long> dp(n+1);
dp[0] = 0; dp[1] = 1;
for (int i = 2; i <= n; i++)
    dp[i] = dp[i-1] + dp[i-2];

Example 2: 0/1 Knapsack

Given N items with weights w[i] and values v[i], fill a bag of capacity W to maximize value. Each item used at most once.

C++
// dp[i][j] = max value using first i items, capacity j
// Transition:
//   Don't take item i: dp[i][j] = dp[i-1][j]
//   Take item i:       dp[i][j] = dp[i-1][j-w[i]] + v[i]

vector<vector<int>> dp(n+1, vector<int>(W+1, 0));

for (int i = 1; i <= n; i++) {
    for (int j = 0; j <= W; j++) {
        dp[i][j] = dp[i-1][j]; // skip item i
        if (j >= w[i])
            dp[i][j] = max(dp[i][j], dp[i-1][j-w[i]] + v[i]);
    }
}
cout << dp[n][W];
Time: O(N·W) Space: O(N·W)

Example 3: Longest Common Subsequence (LCS)

C++
string a, b; cin >> a >> b;
int n = a.size(), m = b.size();
vector<vector<int>> dp(n+1, vector<int>(m+1, 0));

for (int i = 1; i <= n; i++)
    for (int j = 1; j <= m; j++)
        if (a[i-1] == b[j-1])
            dp[i][j] = dp[i-1][j-1] + 1;
        else
            dp[i][j] = max(dp[i-1][j], dp[i][j-1]);

cout << dp[n][m];

Example 4: Longest Increasing Subsequence (LIS) — O(N log N)

C++
vector<int> tails; // tails[i] = smallest tail of LIS of length i+1

for (int x : arr) {
    auto it = lower_bound(tails.begin(), tails.end(), x);
    if (it == tails.end()) tails.push_back(x);
    else *it = x;
}
cout << tails.size(); // LIS length
Time: O(N log N) Naïve DP: O(N²)
Memo vs Tabulation: Top-down (memoization) is easier to code; bottom-up (tabulation) is faster in practice (no recursion overhead). Both give the same answer.
∑ Prefix Sums
Answer range-sum queries in O(1) after O(N) preprocessing

1D Prefix Sum

Build array P where P[i] = A[0] + A[1] + … + A[i-1]. Then sum of A[l..r] = P[r+1] - P[l].

A: [3, 1, 4, 1, 5, 9, 2, 6] P: [0, 3, 4, 8, 9, 14, 23, 25, 31] Sum(A[2..5]) = P[6] - P[2] = 23 - 4 = 19 → 4 + 1 + 5 + 9 = 19 ✓
C++
int n; cin >> n;
vector<int> a(n), p(n+1, 0);
for (int i = 0; i < n; i++) cin >> a[i];

// Build
for (int i = 0; i < n; i++)
    p[i+1] = p[i] + a[i];

// Query sum of a[l..r] (0-indexed, inclusive)
auto query = [&](int l, int r) {
    return p[r+1] - p[l];
};
Build: O(N) Query: O(1)

2D Prefix Sum

For a 2D grid, P[i][j] = sum of rectangle from (0,0) to (i-1, j-1). Query any subrectangle in O(1).

Grid A: Prefix P: ┌───┬───┬───┬───┐ ┌───┬───┬───┬───┐ │ 1 │ 2 │ 3 │ 4 │ │ 0 │ 0 │ 0 │ 0 │ ├───┼───┼───┼───┤ ├───┼───┼───┼───┤ │ 5 │ 6 │ 7 │ 8 │ ──▶ │ 0 │ 1 │ 3 │ 6 │ ├───┼───┼───┼───┤ ├───┼───┼───┼───┤ │ 9 │10 │11 │12 │ │ 0 │ 6 │14 │24 │ └───┴───┴───┴───┘ └───┴───┴───┴───┘ Sum rect (r1,c1)→(r2,c2): = P[r2+1][c2+1] - P[r1][c2+1] - P[r2+1][c1] + P[r1][c1]
C++
int n, m;
cin >> n >> m;
vector<vector<int>> a(n, vector<int>(m));
vector<vector<long long>> p(n+1, vector<long long>(m+1, 0));

for (int i = 0; i < n; i++)
    for (int j = 0; j < m; j++)
        cin >> a[i][j];

// Build 2D prefix
for (int i = 1; i <= n; i++)
    for (int j = 1; j <= m; j++)
        p[i][j] = a[i-1][j-1] + p[i-1][j] + p[i][j-1] - p[i-1][j-1];

// Query sum of rectangle (r1,c1) to (r2,c2), 0-indexed inclusive
auto query = [&](int r1, int c1, int r2, int c2) -> long long {
    return p[r2+1][c2+1]
         - p[r1][c2+1]
         - p[r2+1][c1]
         + p[r1][c1];
};
Build: O(N·M) Query: O(1)
Classic trick: If you need to add 1 to a range [l, r] many times and then read final values — use a difference array: D[l]++, D[r+1]--, then prefix-sum D to get final values. O(N) total.
🌊 BFS — Breadth-First Search
Shortest path in unweighted graphs; explores level by level

How BFS Works

Start from a source. Visit all nodes at distance 1, then distance 2, etc. Uses a queue (FIFO). Guarantees shortest path in unweighted graphs.

Graph: 1──2──4 │ │ 3──5──6 BFS from 1: Level 0: [1] Level 1: [2, 3] Level 2: [4, 5] Level 3: [6] dist: 1→0, 2→1, 3→1, 4→2, 5→2, 6→3

BFS Template

C++
int n; // number of nodes (1-indexed)
vector<vector<int>> adj(n+1); // adjacency list

vector<int> dist(n+1, -1); // -1 = unvisited
queue<int> q;

int src = 1;
dist[src] = 0;
q.push(src);

while (!q.empty()) {
    int u = q.front(); q.pop();

    for (int v : adj[u]) {
        if (dist[v] == -1) { // not visited
            dist[v] = dist[u] + 1;
            q.push(v);
        }
    }
}
// dist[x] = shortest path from src to x (-1 if unreachable)
Time: O(V + E) Space: O(V)

BFS on a 2D Grid

Very common in IOI-style problems: find shortest path from start to end avoiding obstacles.

C++
int n, m;
vector<string> grid(n);
vector<vector<int>> dist(n, vector<int>(m, -1));
queue<pair<int,int>> q;

int dx[] = {0,0,1,-1};
int dy[] = {1,-1,0,0};

dist[sr][sc] = 0;
q.push({sr, sc});

while (!q.empty()) {
    auto [r, c] = q.front(); q.pop();
    for (int d = 0; d < 4; d++) {
        int nr = r + dx[d], nc = c + dy[d];
        if (nr >= 0 && nr < n && nc >= 0 && nc < m
            && grid[nr][nc] != '#'  // not a wall
            && dist[nr][nc] == -1) {
            dist[nr][nc] = dist[r][c] + 1;
            q.push({nr, nc});
        }
    }
}
cout << dist[er][ec]; // shortest path to destination
0-1 BFS: If edges have weight 0 or 1, use a deque. Push weight-0 to front, weight-1 to back. O(V+E) instead of Dijkstra's O(E log V).
🌲 DFS — Depth-First Search
Explore as deep as possible; used for connectivity, cycles, topological sort, tree problems

How DFS Works

Go as deep as possible along one path before backtracking. Uses a stack (or recursion). Does NOT give shortest path (use BFS for that).

Graph: 1──2──4 │ │ 3──5──6 DFS from 1 (one possible order): Visit: 1 → 2 → 4 (backtrack) → 5 → 6 (backtrack²) → 3 Stack: push 1, push 2, push 4, pop 4, push 5, push 6...

Recursive DFS Template

C++
vector<vector<int>> adj(n+1);
vector<bool> visited(n+1, false);

void dfs(int u) {
    visited[u] = true;
    // process u here

    for (int v : adj[u]) {
        if (!visited[v])
            dfs(v);
    }
    // post-process u here (after all children)
}

// Call:
dfs(1);
Time: O(V + E) Space: O(V) stack

Key DFS Applications

Count connected components: Run DFS from each unvisited node, increment counter.

Detect cycles: Track "in current path" with a color (WHITE/GRAY/BLACK).

Topological Sort: After DFS finishes a node, push to stack. Reverse the stack.

Tree: subtree size, depth, LCA: DFS is the go-to.

C++ — Topological Sort
vector<int> order;
vector<bool> visited(n+1, false);

void topoSort(int u) {
    visited[u] = true;
    for (int v : adj[u])
        if (!visited[v]) topoSort(v);
    order.push_back(u); // post-order
}

for (int i = 1; i <= n; i++)
    if (!visited[i]) topoSort(i);

reverse(order.begin(), order.end());
Watch out: Recursive DFS can hit stack overflow for large N (N > 100,000). Either increase stack size or convert to iterative DFS using an explicit stack.

Iterative DFS (safe for large inputs)

C++
stack<int> stk;
stk.push(src);
visited[src] = true;

while (!stk.empty()) {
    int u = stk.top(); stk.pop();
    // process u
    for (int v : adj[u])
        if (!visited[v]) {
            visited[v] = true;
            stk.push(v);
        }
}
01 Binary Conversions
All four conversions + interactive demo

Int → Binary

Use bitset or manual bit extraction.

C++
int n = 42;

// Method 1: bitset (fixed width)
string b = bitset<8>(n).to_string();
// → "00101010"

// Method 2: manual (no leading zeros)
string toBin(int n) {
    if (n == 0) return "0";
    string res;
    while (n > 0) {
        res += (char)('0' + n%2);
        n /= 2;
    }
    reverse(res.begin(), res.end());
    return res;
}

Binary → Int

Use stoi with base 2, or manual shift.

C++
string b = "101010";

// Method 1: stoi
int n = stoi(b, nullptr, 2); // → 42

// Method 2: bitset
int n = (int)bitset<32>(b).to_ulong();

// Method 3: manual
int n = 0;
for (char c : b)
    n = n * 2 + (c - '0');

Str → Binary

Each character → its ASCII value → binary.

C++
string s = "Hi";
string result;
for (char c : s) {
    result += bitset<8>((unsigned char)c).to_string();
    result += ' '; // optional separator
}
// H = 72 → "01001000"
// i = 105→ "01101001"

Binary → Str

Split into 8-bit chunks → each chunk to int → char.

C++
string b = "0100100001101001"; // "Hi"
string result;
for (int i = 0; i < b.size(); i += 8) {
    string chunk = b.substr(i, 8);
    char c = (char)stoi(chunk, nullptr, 2);
    result += c;
}
// result = "Hi"

🎮 Interactive Binary Converter

Bit tricks to remember: n & 1 checks if odd. n >> k checks bit k. n | (1<<k) sets bit k. n & ~(1<<k) clears bit k. __builtin_popcount(n) counts set bits.
🔍 Binary Search
O(log N) search on sorted arrays — and much more

Core Idea

If the answer space is monotone (all FALSEs then all TRUEs, or vice versa), binary search can find the boundary in O(log N). Works on sorted arrays AND on answer spaces.

Array: [1, 3, 5, 7, 9, 11, 13] lo=0 mid=3 hi=6 Target = 9: Step 1: mid=3, A[3]=7 < 9 → lo=4 Step 2: mid=5, A[5]=11 > 9 → hi=4 Step 3: mid=4, A[4]=9 == 9 → FOUND!

Classic Template (find exact value)

C++
// Returns index of target in sorted arr, or -1
int binarySearch(vector<int>& arr, int target) {
    int lo = 0, hi = arr.size() - 1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2; // avoid overflow!
        if      (arr[mid] == target) return mid;
        else if (arr[mid] <  target) lo = mid + 1;
        else                          hi = mid - 1;
    }
    return -1;
}
Time: O(log N) Space: O(1)

STL Shortcuts

C++
// lower_bound: first position where arr[i] >= x
auto it = lower_bound(arr.begin(), arr.end(), x);

// upper_bound: first position where arr[i] > x
auto it = upper_bound(arr.begin(), arr.end(), x);

// Count occurrences of x:
int cnt = upper_bound(arr.begin(), arr.end(), x)
        - lower_bound(arr.begin(), arr.end(), x);

// Check if x exists:
bool found = binary_search(arr.begin(), arr.end(), x);

Binary Search on Answer (VERY common in IOI)

Instead of searching an array, search the answer space. "Is it possible to do X with value ≤ mid?" If this is monotone, binary search on mid.

C++ — Find minimum feasible value
// feasible(x): returns true if answer <= x is achievable
bool feasible(int x) {
    // ... problem-specific check ...
    return true; // or false
}

int lo = MIN_ANSWER, hi = MAX_ANSWER;
while (lo < hi) {
    int mid = lo + (hi - lo) / 2;
    if (feasible(mid))
        hi = mid;     // mid works, try smaller
    else
        lo = mid + 1; // mid doesn't work, need bigger
}
cout << lo; // minimum feasible answer
Golden Rule: Always write mid = lo + (hi - lo) / 2, never (lo + hi) / 2 — it avoids integer overflow when lo + hi exceeds INT_MAX.
📊 Best Subarray (0s and 1s)
Find the subarray with the most 1s and fewest 0s — several interpretations & approaches

Problem Framing

Given a binary array, find a contiguous subarray that is "best" — most 1s relative to 0s. There are multiple valid interpretations depending on the problem statement:

Array: [0, 1, 1, 0, 1, 1, 1, 0] Best by max 1s count alone: [1,1,0,1,1,1] → 5 ones Best by max (1s - 0s): [1,1,1] → score 3 Max consecutive 1s: [1,1,1] → length 3

Approach 1: Maximize (1s − 0s) — Transform to max subarray sum

Replace each 1 → +1 and each 0 → -1. Then find the maximum subarray sum using Kadane's algorithm. This maximizes (ones − zeros).

C++
vector<int> a = {0,1,1,0,1,1,1,0};
int n = a.size();

// Transform: 0→-1, 1→+1
vector<int> b(n);
for (int i = 0; i < n; i++)
    b[i] = (a[i] == 1) ? 1 : -1;

// Kadane's: find max subarray sum
int best = 0, cur = 0;
int bestL = 0, bestR = -1, curL = 0;

for (int i = 0; i < n; i++) {
    cur += b[i];
    if (cur > best) {
        best = cur;
        bestL = curL; bestR = i;
    }
    if (cur < 0) { cur = 0; curL = i + 1; }
}
// best = max (ones - zeros), subarray is a[bestL..bestR]
cout << "Score: " << best << ", from " << bestL << " to " << bestR;
Time: O(N) Space: O(N)

Approach 2: Maximum 1s with at most K zeros allowed (Sliding Window)

Find the longest subarray with at most K zeros. Two-pointer / sliding window.

C++
int K = 1; // allow at most K zeros
int left = 0, zeros = 0, best = 0;
int bestL = 0, bestR = 0;

for (int right = 0; right < n; right++) {
    if (a[right] == 0) zeros++;

    while (zeros > K) {
        if (a[left] == 0) zeros--;
        left++;
    }

    if (right - left + 1 > best) {
        best = right - left + 1;
        bestL = left; bestR = right;
    }
}
// Longest subarray with ≤ K zeros
Time: O(N) Space: O(1)

Approach 3: Prefix Sum — Count 1s in any range quickly

If you need to compare many subarrays, precompute prefix sum of 1s. Then ones(l,r) = P[r+1] - P[l]. Zeros = (r-l+1) - ones.

C++
vector<int> p(n+1, 0);
for (int i = 0; i < n; i++)
    p[i+1] = p[i] + a[i]; // count of 1s

// For subarray [l, r]:
int ones  = p[r+1] - p[l];
int zeros = (r - l + 1) - ones;
int score = ones - zeros;

Approach 4: Maximum consecutive 1s

C++ — One pass
int best = 0, cur = 0;
for (int x : a) {
    cur = (x == 1) ? cur + 1 : 0;
    best = max(best, cur);
}
cout << best;
Which approach to use? The problem wording is key. "Most ones and fewest zeros" most likely means maximize (ones − zeros) → use Kadane's on transformed array. "Longest with ≤ K zeros" → sliding window. "Count ones in ranges" → prefix sum.
